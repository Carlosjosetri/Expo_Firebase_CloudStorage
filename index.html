OBJETIVO

Se creó para facilitar el almacenamiento y la publicación de contenido que el usuario genera. Servicio de almacenamiento de objetos potente, simple y rentable.

¿CÓMO FUNCIONA?

Cloud Storage permite subir y descargar archivos directamente de los clientes. Cloud Storage almacena tus archivos en un depósito de Google Cloud Storage y los hace accesibles a través de Firebase y Google Cloud. Se escala automáticamente, lo que significa que no es necesario migrar a ningún otro proveedor .

FUNCIONES CLAVES

Operaciones robustas: Los SDK de Firebase para Cloud Storage realizan las operaciones de carga y descarga sin importar la calidad de la red. Seguridad sólida: Los SDK de Firebase para Cloud Storage se integran con Firebase Authentication a fin de brindar autenticación intuitiva y sencilla para los programadores. Gran escalabilidad: Está diseñado para escalar a exabytes si tu app se vuelve viral.

1. Integrar los SDK de Firebase para Cloud Storage

implementation 'com.google.firebase:firebase-storage:16.1.0'

2. Crea una referencia

Para que la app pueda tener acceso a los archivos es necesario crear una referencia de estos. Estas referencias se pueden usar para subir o descargar datos, obtener o actualizar metadatos, o borrar el archivo. Una referencia puede apuntar a un archivo específico o a un nodo de nivel superior de la jerarquía. Las referencias se crean con la instancia singleton FirebaseStorage y con su método getReference(). Puedes crear una referencia a una ubicación de un nivel más bajo en el árbol, como 'images/space.jpg' mediante el método getChild() en una referencia existente.

3. Subida

Cuando tengas una referencia, puedes descargar archivos desde Cloud Storage con una llamada a getBytes() o a getStream(). Si prefieres descargar el archivo con otra biblioteca, puedes obtener una URL de descarga con getDownloadUrl().
